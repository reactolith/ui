Slider

An input where the user selects a value from within a given range. Supports native form submission, range sliders with Symfony-compatible field names, auto-submit, and value tooltips on thumbs.

Preview

Usage

<div class="space-y-4 max-w-md">
  <ui-slider json-default-value="[33]"></ui-slider>
  <ui-slider json-default-value="[25, 75]"></ui-slider>
</div>

Value Display (Always)

<!-- Show value indicators above each thumb (always visible) -->
<ui-slider json-default-value="[50]" show-value></ui-slider>

<!-- Also works with range sliders — each thumb shows its value -->
<ui-slider json-default-value="[25, 75]" show-value></ui-slider>

Value Display (Hover Only)

<!-- Value indicators appear only when hovering over the slider -->
<ui-slider json-default-value="[50]" show-value="hover"></ui-slider>

<ui-slider json-default-value="[25, 75]" show-value="hover"></ui-slider>

Single Slider with Form Name

Volume Save 
<!-- Single slider syncs its value to a hidden input named "volume" -->
<ui-slider name="volume" json-default-value="[50]"></ui-slider>

<!-- Submitted form data: volume=50 -->

Range Slider with Symfony-Compatible Names

Price Range Filter 
<!-- Range slider (2 thumbs) auto-creates two hidden inputs:
     price[from]=200 and price[to]=800 (Symfony-compatible) -->
<ui-slider
  name="price"
  json-default-value="[200, 800]"
  json-min="0"
  json-max="1000"
  json-step="10">
</ui-slider>

Range Slider with Custom Field Names

Age Range Search 
<!-- Use nameFrom/nameTo for custom hidden input names -->
<ui-slider
  name-from="age_min"
  name-to="age_max"
  json-default-value="[18, 65]"
  json-min="0"
  json-max="100">
</ui-slider>

<!-- Submitted form data: age_min=18, age_max=65 -->

Auto-Submit on Change

Brightness 
<!-- Auto-submit when the user finishes dragging the thumb -->
<ui-form method="get" action="/filter">
  <ui-form-item name="brightness" auto-submit="onChange">
    <ui-slider name="brightness" json-default-value="[75]"></ui-slider>
  </ui-form-item>
</ui-form>

Range Slider with Auto-Submit and Form Validation

Price Range (€) 
<!-- Range slider with auto-submit.
     Submits price[from] and price[to] when the user stops dragging. -->
<ui-form method="get" action="/products">
  <ui-form-item name="price" auto-submit="onChange">
    <ui-slider
      name="price"
      json-default-value="[100, 500]"
      json-min="0"
      json-max="1000"
      json-step="10">
    </ui-slider>
  </ui-form-item>
</ui-form>

Props

<ui-slider>

PropTypeDefaultDescription
valuenumber | number[]—Controlled value. Single number or array.
defaultValuenumber | number[]—Default value (uncontrolled). Single number or array. Component resets when this changes.
minnumber0Minimum value.
maxnumber100Maximum value.
stepnumber1Step increment.
namestring—Name for form submission. Single slider: uses native Base UI form input. Range slider (2 thumbs): creates hidden inputs named {name}[from] and {name}[to] (Symfony-compatible).
nameFromstring—Explicit name for the 'from' hidden input (range slider). Overrides {name}[from].
nameTostring—Explicit name for the 'to' hidden input (range slider). Overrides {name}[to].
showValueboolean | "always" | "hover"falseShow value indicators above each thumb. true or 'always' = always visible. 'hover' = only visible when hovering over the slider.
disabledbooleanfalseWhether the slider is disabled.
onValueChange(value: number[], thumb: number) => void—Callback fired on every value change (during drag).
onValueCommitted(value: number[], thumb: number) => void—Callback fired when the user finishes dragging.

All components also accept standard HTML attributes (class, id, style, data-*, aria-*, etc.) which are passed through to the underlying element.
